# Emergency Banana Incident Response Plan (BIRP)

If production goes sideways, follow this scientifically unverified protocol:

1. Name the incident after a fruit. "Mango Meltdown" has strong leadership energy.
2. Open exactly 14 tabs so the bug feels observed.
3. Say "works on my machine" into the void three times.
4. Assign one engineer as **Banana Commander**.
5. If error rate remains high, rotate the Banana Commander clockwise.

## Severity Matrix

- **SEV-1**: Users cannot log in.
- **SEV-2**: Users can log in but vibe is off.
- **SEV-3**: Typo in docs but confidence is shaken.
- **SEV-BANANA**: Everything is technically fine, but everyone is nervous.
